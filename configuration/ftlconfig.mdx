## Overview

The `.ftlconfig` file is a JSON configuration file placed in your project root that controls FTL's behavior for that specific project. It is created by `ftl init` and can be edited manually to customize settings.

## File Location

FTL searches for `.ftlconfig` by walking up from the current working directory to parent directories, similar to how Git finds `.git`. This means you can run FTL commands from subdirectories of your project.

```bash
ftl init  # Creates .ftlconfig in current directory
```

## Configuration Hierarchy

Configuration values are merged in the following order (later values override earlier ones):

1. **Built-in defaults** - Hard-coded fallback values
2. **Global config** (`~/.ftl/config.json`) - Set during `ftl setup`
3. **Project config** (`.ftlconfig`) - Project-specific overrides

## Required Fields

<ParamField path="agent" type="string" required>
  The coding agent to use inside the sandbox.
  
  **Valid values:**
  - `"claude-code"` - Claude Code (requires `ANTHROPIC_API_KEY`)
  - `"codex"` - Codex (requires `OPENAI_API_KEY`)
  - `"aider"` - Aider (requires `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`)
  - `"kiro"` - Kiro (requires AWS credentials + browser login)
  
  **Default:** `"claude-code"`
  
  ```json
  {
    "agent": "claude-code"
  }
  ```
</ParamField>

<ParamField path="tester" type="string" required>
  LiteLLM model string for adversarial test generation. The tester runs in parallel with the agent.
  
  **Default:** `"claude-haiku-4-5-20251001"`
  
  **Examples:**
  ```json
  { "tester": "claude-haiku-4-5-20251001" }               // Anthropic direct
  { "tester": "bedrock/us.anthropic.claude-sonnet-4-6" }   // AWS Bedrock
  { "tester": "openai/gpt-4o-mini" }                       // OpenAI
  ```
  
  See [LiteLLM providers](https://docs.litellm.ai/docs/providers) for all supported models.
</ParamField>

## Optional Fields

### Credential Management

<ParamField path="shadow_env" type="array" default="[]">
  Additional environment variable names to shadow beyond what's in `.env`.
  
  These variables are loaded from your host `os.environ`, replaced with shadow values in the sandbox, and tracked for credential swapping.
  
  ```json
  {
    "shadow_env": ["MY_EXTRA_SECRET", "API_TOKEN"]
  }
  ```
</ParamField>

<ParamField path="agent_env" type="array" default="[]">
  Environment variables from your host to forward into the sandbox **without shadowing**.
  
  Use this for non-sensitive configuration like `NODE_ENV` or `LOG_LEVEL`.
  
  ```json
  {
    "agent_env": ["NODE_ENV", "DEBUG"]
  }
  ```
</ParamField>

### Container Setup

<ParamField path="setup" type="string">
  Shell command run once on a **fresh container only**, before the agent starts.
  
  Use this for installing project dependencies. The command runs inside the sandbox at `/workspace`.
  
  ```json
  {
    "setup": "pip install -r requirements.txt 2>/dev/null; npm install --silent 2>/dev/null; true"
  }
  ```
  
  <Note>
  The `true` at the end prevents failure if files are missing. The `setup` command is skipped on warm container reuse — packages persist in `/home/ftl/.local/` across tasks.
  </Note>
</ParamField>

### Snapshot Backend

<ParamField path="snapshot_backend" type="string" default="local">
  Where to store project snapshots.
  
  **Valid values:**
  - `"local"` - rsync to `~/.ftl/snapshots/`
  - `"s3"` - Upload to S3 (requires `s3_bucket`)
  
  ```json
  {
    "snapshot_backend": "s3"
  }
  ```
</ParamField>

<ParamField path="s3_bucket" type="string">
  S3 bucket name for snapshots. **Required** when `snapshot_backend` is `"s3"`.
  
  Snapshots are stored as gzipped tarballs at `s3://<bucket>/snapshots/<project-hash>/<id>.tar.gz`.
  
  ```json
  {
    "snapshot_backend": "s3",
    "s3_bucket": "my-ftl-snapshots"
  }
  ```
</ParamField>

### AWS Integrations

<ParamField path="cloudwatch_log_group" type="string">
  CloudWatch log group for session traces. When set, FTL emits structured JSON events for each session stage.
  
  ```json
  {
    "cloudwatch_log_group": "/ftl/myproject"
  }
  ```
  
  <Tip>
  Use `ftl config --aws` to provision this automatically.
  </Tip>
</ParamField>

<ParamField path="secrets_manager_prefix" type="string">
  AWS Secrets Manager path prefix. When set, FTL fetches secrets from Secrets Manager instead of reading `.env`.
  
  Secrets with JSON object values are expanded into individual keys. Plain-string secrets use the last path segment as the key name (uppercased).
  
  ```json
  {
    "secrets_manager_prefix": "/myproject/prod/"
  }
  ```
</ParamField>

<ParamField path="guardrail_id" type="string">
  Bedrock Guardrail ID for diff safety checking. When set, FTL applies the guardrail to the full diff text before human review, replacing the local lint scan.
  
  If the guardrail intervenes (detects credentials, PII, etc.), the merge is **hard-blocked** and changes are discarded.
  
  ```json
  {
    "guardrail_id": "abc123def456",
    "guardrail_version": "1"
  }
  ```
  
  <Warning>
  Guardrail intervention results in automatic rejection — no manual review.
  </Warning>
</ParamField>

<ParamField path="guardrail_version" type="string" default="DRAFT">
  Bedrock Guardrail version to apply. Used only when `guardrail_id` is set.
  
  ```json
  {
    "guardrail_version": "1"
  }
  ```
</ParamField>

## Complete Example

```json .ftlconfig
{
  "agent": "claude-code",
  "tester": "claude-haiku-4-5-20251001",
  
  "shadow_env": ["MY_EXTRA_SECRET"],
  "agent_env": ["NODE_ENV", "LOG_LEVEL"],
  
  "setup": "pip install -r requirements.txt 2>/dev/null; npm install --silent 2>/dev/null; true",
  
  "snapshot_backend": "s3",
  "s3_bucket": "my-ftl-snapshots",
  "cloudwatch_log_group": "/ftl/myproject",
  "secrets_manager_prefix": "/myproject/prod/",
  "guardrail_id": "abc123def456",
  "guardrail_version": "1"
}
```

## Minimal Example

The smallest valid `.ftlconfig` created by `ftl init`:

```json .ftlconfig
{
  "agent": "claude-code",
  "tester": "claude-haiku-4-5-20251001"
}
```

## Global Config

The file `~/.ftl/config.json` stores machine-wide defaults set during `ftl setup`. Project `.ftlconfig` files override these values.

```json ~/.ftl/config.json
{
  "agent": "claude-code",
  "tester": "claude-haiku-4-5-20251001"
}
```
