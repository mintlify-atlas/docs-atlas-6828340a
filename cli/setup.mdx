## Overview

The `ftl setup` command provides an interactive wizard to configure FTL with everything you need to get started. It guides you through choosing an AI agent, selecting a test generation model, pulling the Docker sandbox image, and configuring your API credentials.

## Usage

```bash
ftl setup
```

## What It Does

The setup wizard walks through these steps:

1. **Docker Check** - Verifies Docker is installed and running
2. **Agent Selection** - Choose your preferred AI coding agent:
   - Claude Code (Anthropic, recommended)
   - Codex (OpenAI)
   - Aider (open-source)
   - Kiro (AWS)
3. **Image Setup** - Pulls or builds the `ftl-sandbox` Docker image
4. **Tester Model** - Select which model to use for test generation:
   - Anthropic API (claude-haiku)
   - AWS Bedrock (claude-sonnet)
   - Skip test generation
5. **API Key Configuration** - Save your ANTHROPIC_API_KEY to `~/.ftl/credentials`

## Image Reconfiguration

If the `ftl-sandbox` image already exists, setup will ask if you want to reconfigure:

```bash
ftl-sandbox image already exists. Reconfigure? [y/N]: n
  Skipping image setup.
```

## Kiro Authentication

When selecting the Kiro agent, you'll see special instructions for browser-based authentication:

```bash
Kiro authentication
  Kiro uses browser-based login. After your first ftl code run,
  authenticate with:
  docker exec -it $(docker ps -qf ancestor=ftl-sandbox) kiro-cli login
  Credentials persist in the container until it is removed.
```

## Fallback Behavior

If pulling from Docker Hub fails, setup automatically falls back to building the image locally:

```bash
  Pulling vvenne/ftl:latest...
  Hub pull failed — building locally (one-time)...
  Image ready.
```

## Next Steps

After running setup:

1. Navigate to your project directory
2. Run `ftl init` to create a `.ftlconfig` file
3. Run `ftl code "your task"` to start coding
